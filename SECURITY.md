# Security Policy

如果你发现了 QeeClaw SDK 中可能影响安全的问题，请不要直接在公开 issue 中披露细节。

## 建议处理方式

请通过私下渠道联系维护团队，并提供：

- 问题描述
- 影响范围
- 复现方式
- 如有可能，提供修复建议

## 公开仓库中的安全约束

公开仓库默认遵循以下原则：

- 不提交真实生产密钥
- 不提交真实生产域名默认值
- 不提交证书、私钥、内部凭证
- 示例默认只使用本地开发地址或占位域名

## 响应原则

维护团队在确认问题后，建议按以下顺序处理：

1. 评估风险等级
2. 确认受影响模块
3. 准备修复
4. 发布修复说明
5. 在合适时间公开披露
