# 许可证说明

当前 SDK 采用“根仓默认许可证 + 子目录例外许可证”策略。

## 根仓默认许可证

- 仓库根目录默认许可证：`Apache-2.0`
- 对应文件：`LICENSE`
- 中文阅读版：`LICENSE.zh-CN.md`

## 各模块许可证

| 模块 | 许可证 |
| --- | --- |
| `qeeclaw-core-sdk` | Apache-2.0 |
| `qeeclaw-product-sdk` | Apache-2.0 |
| `qeeclaw-runtime-sidecar` | Apache-2.0 |
| `meeting_device_firmware` | MIT |

说明：

- 根目录下的大部分文档、脚本和 SDK 包默认受 `LICENSE` 约束
- `LICENSE.zh-CN.md` 仅用于辅助阅读，法律解释与平台识别以 `LICENSE` 为准
- `meeting_device_firmware/` 目录以其自身的 `LICENSE` 为准，作为明确例外
- 各模块分发时，均应保留模块目录下各自的 `LICENSE` 文件
- 如果后续需要统一许可证策略，应在正式发布前完成法律与依赖合规检查
